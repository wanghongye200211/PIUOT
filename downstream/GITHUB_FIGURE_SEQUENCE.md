## GitHub Figure Sequence

This note records a compact four-figure structure that can be rebuilt from the scripts in this repository.

### Figure b. Prediction of Continuous Cell Dynamics
- Goal:
  - show observed cells, predicted rollout cells, and dense trajectory lines in one projected view
- Data export:
  - `piuot/export_trajectory_points.py`
- Main script path:
  - `downstream/figure_b_continuous_dynamics.py`
- Required intermediate:
  - `*_trajectory_points.csv`, generated by `piuot/export_trajectory_points.py`

### Figure c. UMAP Potential Landscape
- Goal:
  - render a selected potential landscape on a 2D coordinate base
  - keep point colors tied directly to a CSV time column
  - provide a standalone time legend for figure assembly
- Typical content:
  - full UMAP potential surface
  - predicted points colored by `0 d, 1 d, 1.5 d, 2 d, 2.5 d, 3 d, 4 d, 5 d`
  - optional M/En terminal marker-check version
- Main script path:
  - `downstream/export_potential_landscape_points.py`
  - `downstream/figure_c_umap_potential_landscape.py`
- Required intermediate:
  - `*_potential_landscape_points.csv`, generated by `export_potential_landscape_points.py`
- Rendering rule:
  - assign colors from CSV `time`
  - do not infer time classes from rendered PNG colors

### Figure D. Benchmark Metric Comparison
- Goal:
  - compare PIUOT against external baselines with readable panels instead of raw metric tables
- Typical content:
  - `W1`
  - `W2^2`
  - `MMD`
  - shared manifold overlay
- Typical use:
  - benchmark figure for GitHub or paper supplements
- Main script path:
  - `downstream/build_figure_d_benchmark_metrics.py`

### Figure e. Warning UMAP Landscape
- Goal:
  - map the warning score back onto the UMAP manifold as a standalone downstream panel
- Typical content:
  - one clean warning panel for GitHub or paper assembly
  - one annotated warning panel with title, axes, and colorbar for inspection
- Color convention:
  - low warning values are intentionally light
  - high warning values are orange-red to dark red and drawn last
- Main script path:
  - `downstream/figure_e_warning_umap.py`

### Figure G. Gene Perturbation Fate Screen
- Goal:
  - show genome-scale in silico single-gene perturbation effects on terminal iPSC fate composition
  - separate the fate-fraction volcano result from gene-expression differential analysis
- Current definition:
  - perturbation is single-gene `z=+10` overexpression in standardized expression space before GAE encoding
  - simulation starts at Day2 and measures final Day5 `M`, `EN`, and `Others` fate fractions
  - each volcano point is one gene perturbation
- Typical content:
  - M/EN fate volcano
  - optional M/EN/Others fate volcano
  - 16-panel stacked trajectory board with unperturbed control plus top 15 perturbation genes
- Main script path:
  - `downstream/figure_g_gene_perturbation_screen.py`

## Design Rule
- Keep the repository code-only.
- Rebuild figures from your current run.
- Do not commit generated outputs into the public repository.
